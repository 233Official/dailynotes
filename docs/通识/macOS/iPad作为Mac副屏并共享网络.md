## 1. 核心诉求与硬件环境

- **核心诉求**：实现 iPad Pro 与 Mac mini 之间的**“零延迟纯有线随航”**，同时让 iPad **通过 USB 线缆共享 Mac 的网络**（包括接管 Surge 代理流量）。
- **硬件拓扑**：
  - 主机：Mac mini (M4 芯片)
  - 终端：iPad Pro 11英寸 (2021款 M1芯片，纯 Wi-Fi 版)
  - 网络环境：Mac 运行 Surge 代理，且背板雷雳接口已挂载高负载外设（TBU405Pro 扩展坞、USB-C 网卡）。

## 2. 踩坑记录与原理解析

在达成最终目标前，经历了三个典型的苹果生态“底层逻辑陷阱”：

### 坑一：线材规格降级与“假有线”随航

- **现象**：使用购买 iPhone 时附赠的普通的白色 Type-C 充电线连接 Mac 背板，随航能亮，但在 Finder（访达）和系统网络中均找不到 iPad。
- **原理**：普通充电线仅有 USB 2.0（480Mbps）的数据传输能力，带宽不足以支撑无损视频流与底层网络握手。此时 macOS 会切断线缆数据，**强制回退到 Wi-Fi 进行无线随航**，导致设备在物理层面上“查无此人”。

### 坑二：主板资源挤兑与雷雳控制器的“断电保护”

- **现象**：换用 80Gbps 的顶级全能线接入 Mac 背板的雷雳接口，iPad 充电图标“绿灯一闪即灭”，底层握手彻底失败。
- **原理**：Mac mini 的背板雷雳总线共享带宽与 PD 供电预算。由于背板已经插满了吃资源的高负载外设（雷雳扩展坞 + 网卡），当接入 M1 芯片的 iPad 并尝试建立 40Gbps + 快充的底层握手时，触发了主板雷雳控制器的**“供电/带宽保护机制 (Power Gating)”**，导致接口被硬件级强制断电锁定。

### 坑三：Wi-Fi 版 iPad 的“隐身网卡”机制

- **现象**：将顶级线缆移至 Mac 前面板（10Gbps USB 3.2 接口）后，硬件识别成功并完成“信任”授权，但在 `系统设置 -> 网络` 列表中依然找不到 `iPad USB`。

  ![image-20260323164948117](http://cdn.ayusummer233.top/DailyNotes/202603231649467.png)

- **原理**：与自带蜂窝基带和“个人热点”的 iPhone 不同，Wi-Fi 版 iPadOS 在 USB 协议下不会主动向 Mac 广播其网络设备身份，从而在常规网络列表中“隐身”。它的虚拟网卡接口仅在 `互联网共享` 的下发目标列表中可见。

  ![image-20260323165500924](http://cdn.ayusummer233.top/DailyNotes/202603231655988.png)

## 3. 最终完美配置方案

为了绕过背板雷雳口的供电死锁，并完美打通数据与网络，最终采用**前面板 10Gbps 通道**方案：

### 步骤一：物理连接与底层信任

1. 准备一根支持 10Gbps 或以上速率的**全功能 Type-C 数据线**（如 80Gbps 全能线）。

2. 将线缆连接至 iPad 和 **Mac mini 的前面板接口**（普通 USB 3.2 控制器，供电与带宽独立）。

3. 解锁 iPad，并在屏幕弹窗中点击**“信任此电脑”**。

4. 打开 Mac 的 `Finder (访达)`，确认左侧边栏能正常显示并读取该 iPad。

   ![image-20260323165617112](http://cdn.ayusummer233.top/DailyNotes/202603231656196.png)

### 步骤二：激活有线随航

1. 在 Mac 控制中心开启随航连接 iPad。

   ![image-20260323165805205](http://cdn.ayusummer233.top/DailyNotes/202603231658308.png)

   如果之前是无线随航的话，插上线后  iPad 屏幕上会出现 **“正在切换到 USB”** 的系统提示，此时纯有线无损视频流通道已打通。

### 步骤三：配置互联网共享与 Surge 接管

1. **断开干扰设备**：暂时拔除前面板可能存在的其他 iOS 设备（如 iPhone 17），避免 `usbmuxd` 进程的虚拟网卡分配冲突。

2. **配置共享路由**：在 Mac 上进入 `系统设置 -> 通用 -> 共享 -> 互联网共享`。

     ![image-20260323165500924](http://cdn.ayusummer233.top/DailyNotes/202603231655988.png)

3. **选定网卡与目标**：

   - `共享以下来源的连接`：选择 **Surge**（或其他主网络接口）。
   - `使用以下端口共享给设备`：在列表中勾选 **`iPad USB`**。

     ![image-20260323165507964](http://cdn.ayusummer233.top/DailyNotes/202603231655018.png)

     ![image-20260323165513376](http://cdn.ayusummer233.top/DailyNotes/202603231655419.png)

4. **开启服务**：打开互联网共享的总开关。

5. **Surge 局域网放行**：确保 Surge 设置中已开启 **“允许局域网访问 (Allow Wi-Fi Access / Gateway)”** 功能，否则底层 NAT 桥接会阻断 iPad 获取 IP。

   ![image-20260323174144214](http://cdn.ayusummer233.top/DailyNotes/202603231741465.png)

6. **终端验证**：关闭 iPad 的 Wi-Fi 开关，测试网页加载。此时 Surge 仪表盘中会出现来源为“未知 (Unknown)”的活跃请求，即为 iPad 成功通过线缆代理上网的流量。

   ![image-20260323170137281](http://cdn.ayusummer233.top/DailyNotes/202603231701376.png)
