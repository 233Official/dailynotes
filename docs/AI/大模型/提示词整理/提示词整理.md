#### Mermaid 转 Drawio

**任务目标**：请将下方的 Mermaid 流程图代码转换为 Draw.io (diagrams.net) 的未压缩 XML 代码。

**核心痛点**：该流程图如果直接生成，会导致图形 [太长(纵向) / 太宽(横向)]，无法在 PPT 或宽屏显示器上完整展示。

**设计要求 (非常重要)**：

1. **布局策略**：请放弃单一方向布局，采用 “折行布局 (Snake Layout / Z-pattern)”。
   - 即：第一行从左到右排列主流程；当遇到长子流程或后续步骤时，通过连线“拐弯”折返到第二行继续排列。
   - 目标是将整体长宽比控制在 16:9 左右，形成一个扁平的矩形视图。
2. **子图处理**：Mermaid 中的 subgraph 请使用 Draw.io 的 Container (Group) 组件包裹，并使用虚线边框，确保逻辑清晰。
3. **样式美化**：
    - **连线**：必须使用 直角连线 (Orthogonal Edge Style)，避免混乱的斜线。
    - **形状语义**：数据库/存储节点请用 圆柱体 (Cylinder)；文档/文件节点请用 文档图标 (Note/Document)；判断节点用 菱形。
    - **配色**：使用专业、柔和的商务配色（如浅蓝、浅紫、浅黄区分不同模块）。
    - **代码输出**：请直接输出完整的 XML 代码块，不要使用压缩格式，以便我复制保存。
